# Contract Mirror UI v5.2.2 Role Corrected Dashboard

## 목적
v5.2에서 사용자 화면과 심사위원 화면의 역할이 뒤집혀 보이던 문제를 수정한다.

## 핵심 원칙
- 사용자 화면(`/`): 계약 현장용 모바일 플로우
- 참여자 화면(`/participant/contractor`, `/participant/explainer`): 모바일 본인확인·녹취 동의
- 심사위원 화면(`/judge`): PC 웹형 검증 대시보드

## 주요 변경
1. 사용자 화면은 다시 모바일 현장 플로우로 고정
2. 심사위원 화면은 모바일 목업 중심이 아니라 검증 대시보드 중심으로 재구성
3. `/judge`에서 Trust Summary, AI 불일치 리포트, 검증 카드 상태가 한 화면에 표시
4. Raw Trust Log는 보조 검증 정보로 유지
5. 결과 리포트의 좌우 비교는 심사위원 웹 대시보드에서 넓게 표시

## 발표 메시지
사용자는 모바일로 계약 현장을 기록하고, 심사위원은 웹에서 그 기록의 증거성과 기술 흐름을 검증한다.
