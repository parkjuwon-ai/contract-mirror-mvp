# Contract Mirror UI v6.6 Step 5 - Mobile Flow Productization

## 목적
v6.6은 현장 입력용 모바일 플로우를 실제 사용자 관점으로 다듬는 단계입니다.
기존 v6.5에서 분리한 `/`, `/report`, `/verify`, `/judge` 구조를 유지하면서 모바일 계약 현장에서 필요한 입력 방식과 사후 확약서 흐름을 추가했습니다.

## 반영 내용

### 1. 계약서 입력 방식 확장
계약서 등록 화면을 단순 파일 업로드에서 다음 3가지 방식으로 확장했습니다.

- PDF 파일 불러오기
- 계약서 링크 불러오기
- 카메라로 연속 스캔

실제 파일/카메라/OCR 연동은 후속 단계에서 붙일 수 있도록 UI와 이벤트 구조를 먼저 분리했습니다.

### 2. 사용자 언어 정리
사용자 화면에서는 세션/해시 같은 개발자 용어의 비중을 낮추고 다음 표현을 중심으로 정리했습니다.

- 안전 계약방
- 계약 확인 기록
- 원본 확인값
- 위변조 방지 기록증

심사위원용 `/judge`와 기술 상세 접힘 영역에서는 Session ID, hash 등 기술 용어를 유지합니다.

### 3. 대기방 상태 관제 강화
계약자/설명자를 두 개의 상태 카드로 구분하고, 설명자 대기 상태에 스피너를 유지해 시스템이 정상적으로 상대방 응답을 기다리는 느낌을 강화했습니다.

### 4. AI 리포트 핵심 키워드 하이라이트
녹취 발언과 계약서 조항에서 실제 충돌하는 단어를 강조했습니다.

- `보장` ↔ `보장하지 않는다`
- `해지` ↔ `위약금`
- `추가로 들어가는 비용` ↔ `별도로 부담`

### 5. 확약서/추가 확인 문서 흐름 추가
AI 분석 결과 후 다음 행동으로 확약서 흐름을 추가했습니다.

- 확약서 요청하기
- 확약서/추가 문서 등록하기
- 검증 카드 생성하기

확약서가 등록되면 `/verify` 검증 카드에 추가 확인 문서로 포함됩니다.
