# Contract Mirror UI v6.8 Action-Oriented Protection UX

v6.8은 v6.7 Field Protection UX 위에 사용자 행동 설계를 강화한 버전입니다.

## 핵심 변경

- 첫 화면 상단의 작은 시작 상태를 제거하고 하단 CTA 중심으로 정리
- 첫 화면 카피에서 `계약 전에 잡아냅니다`를 핵심 약속으로 강조
- 위험 예시의 `불일치 가능성`을 버튼형 칩이 아니라 연결 표시로 변경
- 결과 화면의 `계약 전 확인 질문`을 리포트 상단으로 이동
- 동의/추가 자료/검증 카드 문구를 더 짧고 사용자 중심으로 조정
- 녹취 화면을 `계약서 원본과 연결된 검증 기록`으로 명확화
- 검증 카드 화면은 해시보다 “나중에 쓸 수 있는 기록”의 의미가 먼저 보이도록 조정
- Judge 대시보드는 요약 카드 우선, Raw Log 보조 원칙 유지

## UX 원칙

현장 기록은 모바일, 사후 검토는 리포트, 심사위원 검증은 대시보드입니다.

사용자 모바일 화면에서는 기술 용어보다 행동을 우선합니다.

1. 계약 검증 시작
2. 계약 문서 등록
3. 양측 본인확인 및 동의
4. 계약 설명 녹취
5. AI 불일치 분석
6. 그 자리에서 확인 질문 제시
7. 확약서/추가 문서 등록
8. 검증 카드 생성
