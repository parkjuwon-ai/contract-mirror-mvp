# Contract Mirror UI v6.1 Responsive Foundation Polish

## 목적
v6.0 Step 1의 반응형 기반을 유지하면서, 데모 화면이 “반응형을 못 만든 모바일 뷰”로 오해받지 않도록 마감한다.

## 핵심 수정
- `/` 사용자 화면은 PC에서 스마트폰 미러링 데모처럼 보이도록 폰 프레임을 강제한다.
- 모바일 실제 접속에서는 외부 폰 프레임을 제거하고 실제 모바일 웹처럼 보이게 한다.
- `/report`, `/verify` 성격의 결과/검증 화면은 PC에서 넓은 웹 리포트로 유지한다.
- 결과 화면의 행동 버튼을 “다음 조치” 영역으로 강화한다.
- 사용자 화면의 sample/mock/poc 인상을 줄이고 제품형 문구로 조정한다.
- 모바일 프레임 내부 스크롤바를 숨겨 목업 완성도를 높인다.
- `/judge`는 사용자 폰 프레임 스타일이 침범하지 않도록 대시보드 레이아웃을 고정한다.

## UX 원칙
- 현장 기록은 모바일 웹
- 사후 검증은 웹 리포트
- 심사위원 검증은 Trust Audit Dashboard
- 화면은 제품처럼, 발표 멘트는 정직하게
