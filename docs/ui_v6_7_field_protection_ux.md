# Contract Mirror UI v6.7 Field Protection UX

## 목적
v6.7은 기술 시연 화면을 넘어, 실제 계약 현장에서 계약자가 무엇을 보고 무엇을 눌러야 하는지까지 안내하는 모바일 보호 UX 단계입니다.

## 주요 변경
- 첫 화면 CTA를 하단 고정형 `계약 검증 시작하기`로 강화
- 첫 화면 카피를 `말로 들은 설명과 계약서 조항이 다를 때, 계약 전에 잡아냅니다.`로 압축
- 위험 예시를 `녹취 속 설명 ↔ 계약서 조항` 시각 비교 구조로 변경
- 계약서 등록 후 `추가 자료가 있나요?` 카드 추가
- 동의 화면 문구를 모바일 사용자가 읽기 쉬운 짧은 체크 항목으로 정리
- 녹취 화면에 `현재 녹취는 이 계약서 기록에 연결되어 저장됩니다` 문구와 상태 그리드 추가
- 결과 화면에 `계약 전 확인 질문` 블록 추가
- 결과 후 행동을 `확약서 요청`, `추가 문서 등록`, `검증 카드 생성` 흐름으로 강화
- 사용자 화면은 쉬운 언어를 우선하고, 기술 정보는 /judge 또는 접기 영역으로 분리

## UX 원칙
- 사용자는 기술 용어보다 행동 안내를 먼저 본다.
- AI는 위험을 보여주는 데서 끝나지 않고, 사용자가 계약 전 무엇을 질문해야 하는지 안내한다.
- 확약서·특약·홍보자료·문자 캡처는 계약 설명 후에도 최종 검증 기록에 추가할 수 있다.
