
# Roadmap

## Phase 1: MVP Prototype

Goal: Build a working prototype that demonstrates the core contract explanation verification flow.

Tasks:

* Create FastAPI project structure
* Build landing page
* Build mock identity verification page
* Create contract explanation session flow
* Add contract upload page
* Add transcript input page
* Generate simple AI analysis result page
* Create risk report page

## Phase 2: AI Analysis Improvement

Goal: Improve the quality of inconsistency detection.

Tasks:

* Connect speech-to-text API
* Extract key claims from transcript
* Extract important contract clauses
* Compare verbal claims and contract clauses
* Classify risk level
* Improve prompt structure
* Add evidence-based report generation

## Phase 3: Verification Layer

Goal: Strengthen trust and traceability.

Tasks:

* Generate session hash
* Generate contract document hash
* Generate analysis report hash
* Create QR verification page
* Add DID/blockchain verification mock
* Prepare future integration with real DID infrastructure

## Phase 4: Demo Readiness

Goal: Prepare the MVP for hackathon presentation.

Tasks:

* Polish UI
* Prepare sample scenario
* Add demo data
* Create screenshots
* Connect GitHub repository
* Prepare presentation script
* Record demo flow if needed

## Phase 5: Expansion

Goal: Expand the service beyond the initial MVP.

Possible expansion domains:

* real estate contracts
* insurance contracts
* loan agreements
* investment products
* franchise contracts
* employment contracts

## Long-Term Vision

Contract Mirror aims to become a trusted infrastructure for high-risk contract explanation verification.

The long-term goal is to help users prove not only what was written in the contract, but also what was explained before signing.
